# Document Q&A — RAG System 

A **Retrieval-Augmented Generation (RAG)** system built **from scratch** — without using high-level frameworks like LangChain. Users can upload PDF documents and ask natural language questions, receiving context-aware answers generated by a locally running **Mistral LLM via Ollama**.

This project demonstrates a deep understanding of RAG internals — custom vector store, manual embedding pipeline, FAISS-based similarity search, and a Streamlit frontend — all wired together via a **FastAPI** backend.

---

## Features

- Upload PDF documents via REST API or Streamlit UI
- Ask natural language questions about document content
- Semantic search using a custom FAISS vector store (built from scratch)
- Answer generation using local Mistral LLM via Ollama REST API
- Custom text chunking with configurable size and overlap
- Sentence embeddings using `all-MiniLM-L6-v2` (SentenceTransformers)
- Streamlit frontend for interactive document Q&A
- Input validation — only PDF files accepted
- Standalone test script for pipeline testing without API

---

## Tech Stack

| Layer | Technology |
|---|---|
| Backend API | FastAPI (Python) |
| Frontend UI | Streamlit |
| LLM | Mistral via Ollama (local REST API) |
| Embeddings | SentenceTransformers (`all-MiniLM-L6-v2`) |
| Vector Database | FAISS (`IndexFlatL2`) — custom built |
| PDF Parsing | pypdf (`PdfReader`) |
| Text Chunking | Custom sliding window (chunk\_size=500, overlap=50) |
| Data Validation | FastAPI HTTPException + file type checks |

## How the RAG Pipeline Works

```
User uploads PDF
      │
      ▼
pypdf PdfReader
extracts raw text from all pages
      │
      ▼
Custom chunk_text()
sliding window chunking
(chunk_size=500, overlap=50)
      │
      ▼
SentenceTransformer (all-MiniLM-L6-v2)
converts each chunk → 384-dim vector
      │
      ▼
Custom VectorStore
FAISS IndexFlatL2
stores all chunk vectors + raw text
      │
      ▼
User asks a question
      │
      ▼
Question → SentenceTransformer → 384-dim vector
FAISS.search() → top-3 most similar chunks
      │
      ▼
Retrieved chunks assembled as context
Prompt engineered with strict instructions
      │
      ▼
Ollama REST API (Mistral)
POST http://localhost:11434/api/generate
      │
      ▼
Answer returned to user
```

## API Endpoints

### `POST /upload-pdf`
Upload a PDF document to be processed and indexed.

**Request:** `multipart/form-data`

| Field | Type | Description |
|---|---|---|
| file | File | PDF file only (validated) |

**Response:**
```json
{
  "message": "PDF uploaded and processed successfully.",
  "total_chunks": 42
}
```

**Error (non-PDF file):**
```json
{
  "detail": "Only PDF files are allowed."
}
```

---

### `POST /ask`
Ask a natural language question about the uploaded document.

**Query Parameter:**

| Parameter | Type | Description |
|---|---|---|
| question | string | Natural language question |

**Response:**
```json
{
  "question": "What is this document about?",
  "answer": "The document discusses..."
}
```

**Error (no document uploaded):**
```json
{
  "detail": "No document uploaded yet."
}
```

## Installation & Setup

### Prerequisites
- Python 3.10+
- [Ollama](https://ollama.com) installed and running locally
- Mistral model pulled in Ollama

### 1. Pull Mistral model in Ollama
```bash
ollama pull mistral
```

### 2. Clone the repository
```bash
git clone https://github.com/yourgithub/document-qna-rag.git
cd document-qna-rag
```

### 3. Create and activate virtual environment
```bash
python -m venv venv
source venv/bin/activate        # Linux / Mac
venv\Scripts\activate           # Windows
```

### 4. Install dependencies
```bash
pip install -r requirements.txt
```

### 5. Run the FastAPI backend
```bash
uvicorn app.main:app --reload
```

### 6. Run the Streamlit frontend (new terminal)
```bash
streamlit run frontend/app.py
```

### 7. Open Swagger UI
```
http://localhost:8000/docs
```

## Testing the Pipeline (Without API)

A standalone test script is included to test the full RAG pipeline directly:

```bash
python app/test_rag.py
```

This will:
- Load `data/sample.pdf`
- Chunk the text
- Generate and store embeddings in FAISS
- Run a test query
- Print top retrieved chunks and the final LLM answer
